# GitLaw Pro 90-Day Sprint Board

## Goal

In 90 Tagen von "starke Beta" zu "erste bezahlte Kanzlei-Piloten auf realem Workflow".

## Phase 1 — Market + offer (Days 1-14)

### Sprint goal

The company has a clear pitch, a clear pilot offer, and a target list.

### Tasks

- finalize company thesis
- finalize pilot offer and pricing
- define ICP list
- create outreach list of 20 target firms
- prepare founder sales deck / short demo script
- define one canonical demo story

### Deliverables

- pilot offer doc
- 20-firm target list
- 1 demo script
- 1 outreach template

## Phase 2 — Product truth and instrumentation (Days 15-30)

### Sprint goal

The killer workflow is measurable and demo-safe.

### Tasks

- instrument intake -> case -> research -> draft funnel
- define usage events
- tighten onboarding copy
- tighten pricing page around pilot logic
- ensure Bao/Nguyen flow is demo-stable
- identify biggest demo failure points

### Deliverables

- funnel metrics spec
- stable demo account / preset
- founder checklist for live demo

## Phase 3 — Production foundation gap close (Days 31-50)

### Sprint goal

The product becomes safe enough for limited real-case pilot usage.

### Tasks

- auth baseline
- tenant model verification
- server-side RBAC review
- secure session handling
- audit event review
- storage architecture decision

### Deliverables

- production gap list with owners
- chosen auth/storage path
- list of must-close trust items before first real pilot

## Phase 4 — Document wedge (Days 51-70)

### Sprint goal

Documents become the strongest reason to buy.

### Tasks

- secure upload flow
- document object model
- OCR pipeline v1
- translation marker / review path
- case document timeline refinement
- original / OCR / translated / approved states

### Deliverables

- working document-to-case pipeline
- review-ready document screen
- one end-to-end migration-heavy sample matter

## Phase 5 — Paid pilot conversion (Days 71-90)

### Sprint goal

At least one first paid pilot, or a very near-close pipeline with live pilot readiness.

### Tasks

- run firm demos
- collect objections
- refine pilot packaging
- push first paid commitment
- define onboarding schedule for first partner
- record baseline time-savings claims

### Deliverables

- 1 paid pilot or signed start path
- objection log
- pilot onboarding checklist

## Weekly operating rhythm

### Monday

- product + revenue review
- decide one core workflow improvement

### Tuesday-Thursday

- build / tighten / test
- firm conversations or demos

### Friday

- review learnings
- update objection log
- update next week's sprint target

## Non-negotiables

- no broad new feature tracks unless they support the main loop
- no enterprise-only work before pilot proof
- no new practice-area sprawl unless demanded by pilot firms
- no generic AI additions without workflow justification

## Metrics to review every week

- number of firm conversations
- demos completed
- pilot interest level
- time-to-first-draft in demo flow
- research save / approve rate
- number of document jobs processed

## 90-day exit condition

At the end of 90 days, one of these must be true:

1. one paid pilot is running
2. two pilot firms are in final onboarding
3. the product truth says the wedge is wrong and must be narrowed

Anything else means the company is still not focused enough.
